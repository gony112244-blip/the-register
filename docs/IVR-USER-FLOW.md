# מערכת טלפונית IVR — זרימת שיחה מלאה

מסמך זה מתאר מה המשתמש שומע בכל שלב, מבוסס על `ivr/router.js`.  
גרסאות זכר/נקבה מטופלות בקוד — כאן מוצגת גרסת זכר כברירת מחדל.

---

## 1. בדיקות כניסה (לפני התפריט)

| מצב | מה נשמע |
|-----|---------|
| מספר לא רשום | מספר הטלפון אינו רשום במערכת הפנקס. להרשמה, יש להיכנס לאתר פינקס. |
| חשבון חסום | החשבון שלך חסום. לפרטים, פנה לצוות התמיכה דרך האתר. |
| פרופיל לא מאושר | הפרופיל שלך עדיין ממתין לאישור. נעדכן אותך במייל כאשר יאושר. |

## 4. תפריט ראשי — `menu`

**מה נשמע (זכר):**

> להצעות חדשות, הקש אחת. לבדיקות התאמה שהגיעו אליך, הקש שתיים. לסטטוס הפניות שלך, הקש שלוש. לניהול תמונות, הקש ארבע. להודעות חשובות, הקש חמש. לכל ההצעות כולל ישנות, הקש שש. להגדרות, הקש תשע. לתמיכה, הקש אפס.

**מקשים:**

| מקש | מה קורה |
|-----|---------|
| 1 | הצעות חדשות |
| 2 | בדיקות התאמה נכנסות |
| 3 | סטטוס פניות שיצאו ממני |
| 4 | בקשות תמונה |
| 5 | הודעות חשובות |
| 6 | כל ההצעות (כולל ישנות / שדולגו) |
| 9 | הגדרות — החלפת PIN |
| 0 | לתמיכה ולעזרה, יש להיכנס לאתר פינקס. (ניתוק אחרי) |
| אחר | חזרה לתפריט עם סטטוס מעודכן |

**קידומות אפשריות לפני תפריט** (כשרשימה ריקה):
- אין הצעות זמינות כרגע.
- אין בדיקות התאמה ממתינות לתשובתך.
- אין פניות פעילות שיצאו ממך.
- אין בקשות תמונה ממתינות.
- אין הודעות חדשות הדורשות תשומת לבך.
- סיימת את כל ההצעות החדשות. לעיון בכל ההצעות לחץ שש בתפריט הראשי.

## 5. הצעות חדשות (`matches`) / כל ההצעות (`all_matches`)

### שכבה 1 — פרטי בסיס (תמיד)

> הצעה. {שם}, {סטטוס אישי}, גיל {בעברית}, מ{עיר}, ישיבת X / בוגרת סמינר X.

אם אין פרטים: הצעה. פרטים לא זמינים.

### אחרי ההצעה — מקשים (זכר):

> הקש אחת — מעוניין. הקש שתיים — לא מעוניין. הקש שמונה — דלג. הקש ארבע לפרטים נוספים. הקש חמש לתיאור מלא. הקש אפס לתפריט הראשי.

### מקש 4 — פרטים נוספים (שכבה 2)

מוצא, רקע משפחתי, עיסוק, גובה (בסנטימטר), כיסוי ראש (לנשים).  
אם ריק: אין פרטים נוספים.  
אחריו (זכר): הקש חמש לתיאור מלא. הקש אחת — מעוניין. הקש שתיים — לא מעוניין. הקש שמונה — דלג. הקש אפס לתפריט.

### מקש 5 — תיאור מלא (שכבה 3)

מבנה גוף, מראה, גוון עור, שאיפה, תחום עבודה, about_me.  
אם ריק: אין תיאור נוסף.  
אחריו: הקש אחת — מעוניין. הקש שתיים — לא מעוניין. הקש שמונה — דלג. הקש אפס לתפריט.

### תוצאות פעולה (קידומת לפני ההצעה הבאה)

| מקש | מה נשמע |
|-----|---------|
| 1 — שליחת פנייה | פנייתך נקלטה. / אירעה תקלה בשליחת הפנייה. |
| 2 — הסרה | הצעה הוסרה. |
| 8 — דלג | (שקט) |
| 0 | חזרה לתפריט |

**סיום הרשימה:**
- הצעות חדשות: סיימת את כל ההצעות החדשות. לעיון בכל ההצעות לחץ שש בתפריט הראשי.
- כל ההצעות: סיימת את כל ההצעות הזמינות.

**מקש לא מוכר:** מקש לא מוכר. + חזרה על הוראות המקשים.

## 7. פניות שיצאו — `my_sent`

### 7א. פנייה ממתינה (`pending`)

> הקש אחת לביטול הפנייה. הקש שמונה להמשך. הקש אפס לתפריט הראשי.

**אחרי ביטול (1):** הפנייה בוטלה. עוברים לפנייה הבאה.

### 7ב. פנייה פעילה (`active`)

> הקש שמונה להמשך. הקש אפס לתפריט הראשי.

**דילוג 8:** עוברים לפנייה הבאה.

## 10. הגדרות — החלפת PIN (`settings`)

| שלב | מה נשמע |
|-----|---------|
| כניסה (מקש 9) | להחלפת קוד הכניסה, הקש את הקוד הנוכחי, ארבע ספרות. |
| timeout / חזרה | אנא הקש את קוד הכניסה הנוכחי. |
| קוד נוכחי נכון | קוד נכון. הקש קוד חדש בן ארבע ספרות. |
| קוד שגוי (עד 3) | קוד שגוי. נסה שוב. הקש את קוד הכניסה הנוכחי. |
| 3 שגיאות | קוד שגוי יותר מדי פעמים. → חזרה לתפריט |
| חסימה | הכניסה נחסמה זמנית עקב ניסיונות שגויים. נסה שוב מאוחר יותר. |
| קוד חדש לא תקין | קוד לא תקין. הקש ארבע ספרות בדיוק. |
| אישור קוד חדש | הקש שוב את הקוד החדש לאישור. |
| קודים לא תואמים | הקודים אינם תואמים. הקש קוד חדש בן ארבע ספרות. |
| עדכון הצליח | הקוד עודכן בהצלחה. → חזרה לתפריט |
| שגיאת מסד נתונים | אירעה תקלה בעדכון הקוד. נסה שוב מאוחר יותר. |

## 12. הערה טכנית למפתח

- `yemotRead` שולח לימות פקודת `read` עם הפרמטר האחרון `no` — המשמעות: **ללא דרישת סולמית (#) לאישור** לאחר כל הקשה. המשתמש לוחץ ספרה ומיד מקבל תגובה.
- הקוד `function yemotRead(...)` הוא **JavaScript** שחי בקובץ `ivr/router.js`. **לא** מדביקים אותו ב-bash.
- כדי לפרוס שינוי: עריכה בפרויקט → `scp ivr/router.js root@pinkas.cloud:/var/www/hapinkas/ivr/router.js` → `pm2 restart hapinkas`.
