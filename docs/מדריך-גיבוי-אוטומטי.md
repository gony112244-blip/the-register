# מדריך הגדרת גיבוי אוטומטי לענן — הפנקס

**מה המדריך הזה עושה:**
- מגדיר גיבוי **יומי אוטומטי** של מסד הנתונים בשרת
- שולח את הגיבוי ל-**Google Drive** (חינמי, 15 GB)
- שומר **30 ימים** של גיבויים, ומוחק ישנים אוטומטית

**זמן הגדרה חד-פעמי:** כ-20 דקות

---

## שלב 1 — כנס לשרת

```bash
ssh root@pinkas.cloud
```

---

## שלב 2 — העבר את סקריפט הגיבוי לשרת

בפעם הבאה שתעשה `git pull` בשרת הסקריפט `backup_to_cloud.sh` יגיע אוטומטית.  
לאחר מכן הפוך אותו להרצה:

```bash
cd /var/www/hapinkas
chmod +x backup_to_cloud.sh
```

---

## שלב 3 — בדוק שהגיבוי עובד (ידנית פעם ראשונה)

```bash
cd /var/www/hapinkas
bash backup_to_cloud.sh
```

תראה משהו כזה:
```
═══════════════════════════════════
  🔄 גיבוי מסד נתונים — הפנקס
  📅 2026-04-08_02-00-00
═══════════════════════════════════
✅ גיבוי נוצר: backup_2026-04-08_02-00-00.sql (150K)
⚠️  rclone לא מותקן — גיבוי נשמר מקומית בלבד
```

בשלב זה הגיבוי עובד מקומית. ממשיך לשלב 4 להוסיף ענן.

## שלב 4 — התקן rclone (כלי העלאה לענן)

```bash
curl https://rclone.org/install.sh | bash
```

בדוק שהותקן:
```bash
rclone --version
```

## שלב 5 — חבר rclone ל-Google Drive

**הפקודה:**
```bash
rclone config
```

**תפריט אינטרקטיבי יופיע — ענה כך:**

| שאלה | מה לכתוב |
|------|----------|
| `n) New remote` | הקש `n` ואחר כך Enter |
| `name>` | כתוב `gdrive` ← חשוב! בדיוק ככה |
| `Storage>` | חפש Google Drive ברשימה, הקש את המספר שלו (בדרך כלל `18` — בדוק בפועל) |
| `client_id>` | Enter (ריק) |
| `client_secret>` | Enter (ריק) |
| `scope>` | הקש `1` (Full access) |
| `root_folder_id>` | Enter (ריק) |
| `service_account_file>` | Enter (ריק) |
| `Edit advanced config?` | הקש `n` |
| `Use auto config?` | הקש `n` ← חשוב! (שרת ללא דפדפן) |

**בנקודה הזו rclone ייתן לך קישור — כך תאשר:**

1. **העתק את הקישור** שמופיע בטרמינל
2. **פתח אותו במחשב שלך** (לא בשרת) בדפדפן רגיל
3. **התחבר לחשבון Google** שאתה רוצה לשמור שם את הגיבויים
4. **אשר גישה** ל-rclone
5. **העתק את הקוד** שגוגל מציגה
6. **חזור לשרת** והדבק את הקוד

לאחר מכן:

| שאלה | מה לכתוב |
|------|----------|
| `Configure this as a Shared Drive?` | הקש `n` |
| `Keep this remote?` | הקש `y` |
| `q) Quit config` | הקש `q` |

## שלב 7 — הרץ גיבוי מלא עם ענן (בדיקה)

```bash
cd /var/www/hapinkas
bash backup_to_cloud.sh
```

הפעם תראה גם:
```
☁️  מעלה ל-Google Drive...
✅ הועלה לענן בהצלחה
```

בדוק ב-Google Drive שלך — תיקייה בשם `hapinkas-backups` עם קובץ `.sql`.

---

## שלב 8 — הגדר Cron (גיבוי אוטומטי יומי)

```bash
crontab -e
```

אם שואל על עורך — בחר `nano` (הפשוט ביותר).

**הוסף שורה זו בסוף הקובץ:**

```
0 2 * * * /bin/bash /var/www/hapinkas/backup_to_cloud.sh >> /var/www/hapinkas/logs/backup.log 2>&1
```

**מה זה אומר:**
- `0 2 * * *` — כל לילה ב-02:00 בדיוק
- `>> ...backup.log` — שומר לוג של כל גיבוי (הצלחה/שגיאה)

**שמור וצא:** `Ctrl+X` → `Y` → Enter

---

## שלב 9 — צור תיקיית לוגים

```bash
mkdir -p /var/www/hapinkas/logs
```

---

## בדיקת תקינות (בעתיד)

**לראות לוג הגיבויים:**
```bash
tail -50 /var/www/hapinkas/logs/backup.log
```

**לראות גיבויים מקומיים:**
```bash
ls -lh /var/www/hapinkas/backups/
```

**לראות גיבויים בענן:**
```bash
rclone ls gdrive:hapinkas-backups/
```

## שחזור מגיבוי (אם יש צורך — חס ושלום)

```bash
# הורד מהענן (אם צריך)
rclone copy gdrive:hapinkas-backups/backup_2026-04-08_02-00-00.sql /tmp/

# שחזר לDB
psql "$DATABASE_URL" < /tmp/backup_2026-04-08_02-00-00.sql
```

או עם פרמטרים נפרדים:
```bash
psql -h 127.0.0.1 -U postgres -d hapinkas_db < /tmp/backup_XXXX.sql
```
