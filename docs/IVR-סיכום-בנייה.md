# מערכת IVR "הפנקס" — סיכום מה שנבנה ומה שנשאר

**תאריך:** אפריל 2026  
**ספק תקשורת:** ימות המשיח  
**מנוע TTS:** Google Cloud Text-to-Speech (`he-IL-Wavenet-B`, קול גברי)  
**שרת:** Node.js + Express | DB: PostgreSQL

---

## מה נבנה — רשימה מלאה

### שכבת תשתית

| קובץ | תיאור |
|------|-------|
| `ivr/auth.js` | אימות token של ימות משיח, חיפוש משתמש לפי מספר טלפון, בדיקת PIN (bcrypt), ניהול sessions ב-DB, עדכון PIN |
| `ivr/tts.js` | יצירת MP3 דרך Google Cloud TTS, Cache מקומי לפי hash + שם קול (מניעת יצירה כפולה), הבחנה בין `static` ו-`dynamic` |
| `ivr/data.js` | כל שאילתות ה-DB הייחודיות ל-IVR (ספירות, שליפות, פעולות) |
| `ivr/router.js` | מנהל הזרימה המרכזי — State Machine שמטפל בכל מצב של השיחה |
| `ivr/phonetic-map.json` | מילון הגייה לשיפור ניסוח בעברית (ערים, ישיבות, סטטוסים) |

---

### זרימת כניסה ואימות

- **זיהוי לפי Caller ID** — חיפוש מספר הטלפון ב-DB
- **משתמש לא רשום** → הקראת הפניה לאתר
- **משתמש חסום / לא מאושר** → הקראת הסבר והפניה
- **PIN** — בקשת 4 ספרות, אימות מול hash bcrypt ב-DB
- **3 ניסיונות שגויים** → חסימה ל-30 דקות
- **כניסה ללא PIN** (`allow_ivr_no_pass = true`) → מעבר ישיר לתפריט
- **מבזק סטטוס** בכניסה: "יש לך X הצעות, Y פניות, Z בקשות תמונה"

---

### תפריט ראשי ומקשים

| מקש | פיצ'ר | פעולות אפשריות |
|-----|-------|----------------|
| **1** | הצעות חדשות (matches) | הקשבה, מקש 4 = פרטים נוספים, מקש 5 = תיאור מלא, 1 = שלח פנייה, 2 = לא מעוניין, 8 = דלג |
| **2** | פניות נכנסות ממתינות | מקש 4 = פרטים, מקש 5 = תיאור מלא, 1 = אשר, 2 = דחה, 8 = דלג |
| **3** | סטטוס פניות שיצאו ממני | הקשבה לפי סטטוס (pending/active/waiting_for_shadchan), 1 = בטל פנייה ממתינה, 8 = דלג |
| **4** | ניהול תמונות | מי מבקש לראות את תמונתי, 1 = אשר (+ אישור הפוך אוטומטי), 2 = דחה, 8 = דלג |
| **5** | הודעות חשובות (מסוננות) | הקשבה, 1 = האזן שוב, 8 = הודעה הבאה (+ סימון כנקראה) |
| **9** | הגדרות — שינוי PIN | אימות PIN נוכחי → הזנת חדש → אישור → שמירה מוצפנת |
| **0** | תמיכה | הפניה לאתר הפנקס |
| **#** | חזרה לתפריט ראשי | מכל שלב בשיחה |

### לוגיקה מיוחדת שנבנתה

- **פנייה מגדרית** — כל הטקסטים מותאמים לגבר/אישה לפי `users.gender`
- **3 שכבות פרטים** לכל פרופיל (הצעות + פניות נכנסות):
  - שכבה 1 (ברירת מחדל): שם, גיל, עיר, מוסד לימודים
  - שכבה 2 (מקש 4): מוצא, רקע משפחתי, עיסוק, גובה, כיסוי ראש
  - שכבה 3 (מקש 5): מראה, גוף, שאיפות, תחום עבודה, "על עצמי"
- **Cache TTS חכם** — כל MP3 שנוצר נשמר; קבצים סטטיים (תפריטים) לעולם, קבצים דינמיים (פרופילים) לפי hash
- **Pagination** — כל תת-תפריט עובד בלולאה עם `page offset`
- **אישור תמונה הפוך** — מי שמאשר חשיפת תמונה אוטומטית מקבל גישה גם הוא
- **סינון הודעות** — רק הודעות רלוונטיות: מהשדכנית, ביטולי פנייה, תגובות לבקשות תמונה

|------|--------------|

## מה נשאר לעשות

### קריטי לפני השקה

| משימה | פירוט |
|-------|-------|
| **חיבור ימות המשיח** | רכישת מספר קבוע, הגדרת ה-webhook בלוח הניהול של ימות → כתובת: `https://pinkas.cloud/ivr/call?token=...` |
| **בדיקה עם שיחת טלפון אמיתית** | חיוג למספר וסיור מלא בכל המקשים עם פרופיל אמיתי |
| **ניקוי קבצי אודיו ישנים** | `rm -rf /var/www/hapinkas/ivr-audio/static/* /var/www/hapinkas/ivr-audio/dynamic/*` לפני השקה (הקובץ הוקלט בקול שונה) |

### שיפורים שאפשר להוסיף בשלב הבא

| משימה | עדיפות |
|-------|--------|
| **אייקון "נשמע בטלפון"** באתר ליד הצעות שנשמעו (`ivr_match_views`) | בינונית |
| **שליחת מייל אוטומטי** לצד השני בעת אישור/דחייה מה-IVR (כמו שיש באתר) | בינונית |
| **ביטול חיבור active** דרך ה-IVR (כרגע רק ביטול pending) | נמוכה |
| **לוג שיחות** (`ivr_logs`) — כמה זמן שהה, באיזה מקש יצא | נמוכה |
| **Timeout חוזר** — חזרה אוטומטית על האפשרויות אחרי 8 שניות ללא הקשה | נמוכה |
| **שליחת בקשת תמונה** מה-IVR (לצד עם חיבור active) | נמוכה |

## ארכיטקטורה — זרימה טכנית

```
מתקשר חייג
     │
     ▼
ימות המשיח ← שולח GET ל-webhook
     │         /ivr/call?phone=...&digits=...&EnterID=...&token=...
     ▼
ivr/router.js
  ├── אימות token
  ├── חיפוש משתמש לפי phone
  ├── טעינת/יצירת session (לפי EnterID + user_id)
  ├── State Machine:
  │     init → waiting_pin → menu → [matches / requests / my_sent / photos / messages / settings]
  └── בניית טקסט → textToUrl() → Cache → Google TTS → MP3
             │
             ▼
      JSON לימות המשיח:
      { action: 'read'|'playback'|'hangup', file: 'https://...mp3', numDigits: N }
             │
             ▼
      ימות המשיח מנגן את ה-MP3 למתקשר
```
