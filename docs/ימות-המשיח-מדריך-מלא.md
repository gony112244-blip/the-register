## 1. מה זה ימות המשיח

**ימות המשיח** הוא ספק שירותי תקשורת מוכוון לקהל החרדי-דתי בישראל. הם מספקים:

- **קווי טלפון** תואמי "טלפון כשר" (ללא אינטרנט)
- **מערכת IVR** (Interactive Voice Response — מענה קולי אוטומטי)
- **API** לחיבור שרת חיצוני לניהול השיחות
- **אחסון קבצי אודיו** בשרתים שלהם
- **ניתוב שיחות** (Routing)

**מה שהם לא:**  
ימות המשיח **אינם שרת** — הם **לקוח**. כאשר מישהו מחייג, ימות שולחים **GET request** לשרת שלנו, ואנחנו שולטים בשיחה דרך התגובות שאנחנו מחזירים.

**אתר:** [www.call2all.co.il](https://www.call2all.co.il)  
**תיעוד API:** [https://f2.freeivr.co.il/post/75](https://f2.freeivr.co.il/post/75)

## 4. API — תיעוד מלא

### Base URL
```
https://www.call2all.co.il/ym/api
```

### 4.1 Login — קבלת Token לסשן

```
GET /Login?username=0772291821&password=68848555
```

**תגובה:**
```json
{
  "responseStatus": "OK",
  "token": "eyJhbGci..."
}
```

**כללים:**
- Token תקף **30 דקות** בלבד
- כדאי לרענן כל **25 דקות** (buffer של 5 דקות)
- שם משתמש = **מספר הטלפון** של הקו (לא כתובת מייל)

**קוד Node.js — ניהול Token אוטומטי:**
```js
let sessionToken = null;
let tokenExpiry = 0;

async function getToken() {
    if (sessionToken && Date.now() < tokenExpiry) return sessionToken;
    const { data } = await axios.get(`${BASE_URL}/Login`, {
        params: { username: process.env.YEMOT_NUMBER, password: process.env.YEMOT_PASSWORD }
    });
    if (data.responseStatus !== 'OK') throw new Error('Yemot login failed');
    sessionToken = data.token;
    tokenExpiry = Date.now() + 25 * 60 * 1000; // 25 דקות
    return sessionToken;
}
```

### 4.2 UploadFile — העלאת קובץ אודיו

```
POST /UploadFile
Content-Type: multipart/form-data

Fields:
  token=<token מ-Login>
  path=ivr2:/folder/subfolder/filename.wav
  convertAudio=1
  file=<binary content>
```

**פרמטרים חשובים:**

| פרמטר | ערך | הסבר |
|--------|-----|-------|
| `token` | מה שקיבלנו מ-Login | חובה |
| `path` | `ivr2:/xxx/yyy/name.wav` | נתיב הקובץ בשרתי ימות |
| `convertAudio` | `1` | **חובה!** ימות ממירים MP3 ל-WAV אוטומטית |
| `file` | binary | תוכן הקובץ |

> ⚠️ **convertAudio=1 הוא חובה מוחלטת!** אם מעלים MP3 בלי זה — הקובץ לא יושמע. ימות עובדים עם WAV פנימית.

**קוד Node.js:**
```js
const FormData = require('form-data');
const fs = require('fs');

async function uploadFile(localPath, yemotPath) {
    const token = await getToken();
    const form = new FormData();
    form.append('token', token);
    form.append('path', `ivr2:${yemotPath}`);  // ivr2:/tts_cache/2a/hash.wav
    form.append('convertAudio', '1');
    form.append('file', fs.createReadStream(localPath));
    
    const { data } = await axios.post(`${BASE_URL}/UploadFile`, form, {
        headers: form.getHeaders()
    });
    return data;
}
```

## 5. פרוטוקול Webhook — איך ימות מתקשרים אלינו

### 5.1 הזרימה הכללית

```
מתקשר חייג → ימות המשיח מקבלים שיחה
                    ↓
    GET https://pinkas.cloud/ivr/call?token=...&ApiPhone=054...&digits=...
                    ↓
          השרת שלנו מחזיר תגובת טקסט
                    ↓
       ימות מנגנים קובץ / מבקשים הקשה / מנתקים
                    ↓
          (לחיצת מקש → GET חדש עם digits=X)
```

### 5.2 פרמטרים שימות שולחים לנו

| פרמטר | תיאור | דוגמה |
|--------|-------|-------|
| `ApiPhone` | מספר המתקשר | `0541234567` |
| `ApiCallId` | מזהה ייחודי לשיחה | `abc123-def456` |
| `ApiYFCallId` | גרסה עדיפה של מזהה השיחה | `yfcall-abc-123` |
| `ApiDID` | המספר שאליו חויגו | `0772291821` |
| `ApiRealDID` | המספר האמיתי (אחרי ניתוב) | `023130778` |
| `ApiExtension` | השלוחה הנוכחית | `1` |
| `ApiTime` | Unix timestamp | `1712345678` |
| `ApiHangupExtension` | מגיע רק בסיום שיחה | `0` |
| `hangup` | האם המתקשר ניתק | `yes` / ריק |
| `digits` | מה הלקוח הקיש | `1`, `42`, ריק |
| `token` | הטוקן שהגדרנו ב-URL | `hapinkas-ivr-secret-2026` |

### 5.3 מה חשוב לדעת על הפרמטרים

**`ApiYFCallId` vs `ApiCallId`:**
- שניהם קיימים, `ApiYFCallId` עדיף — הוא יותר יציב
- בדוק את שניהם: `const enterId = req.query.ApiYFCallId || req.query.ApiCallId`
- **קבוע לאורך כל השיחה** — משמש כ-session ID

**`digits` — תמיד נרמל!**
```js
const key = digits != null ? String(digits).trim() || null : null;
```
ימות יכולים לשלוח מחרוזת, מספר, או ריק. ללא נרמול → `TypeError: digits?.trim is not a function`

**`hangup=yes`:**
- ימות שולחים בקשה נוספת כשהמתקשר ניתק
- חשוב לטפל: לנקות session ולהחזיר תגובה ריקה
```js
if (req.query.hangup === 'yes') {
    // cleanup session
    return res.type('text').send('');
}
```

## 6. פורמט תגובות לימות

ימות מצפים לתגובה **טקסטית** (לא JSON!). פורמט `key=value`, שורה אחת.

### 6.1 השמעה בלבד (ללא הקשה)

```
id_list_message=f-/tts_cache/2a/hash
```

### 6.2 השמעה + בקשת הקשה (Read)

```
read=f-/tts_cache/2a/hash=digits,,1,1,8,NO,no,yes
```

פירוט הפרמטרים לפי סדר:

```
read=<segment_audio>=<var_name>,,<max_digits>,<min_digits>,<timeout_sec>,NO,no,<allow_empty>
```

| מיקום | ערך | משמעות |
|--------|-----|--------|
| 1 | `f-/path/to/audio` | הקובץ להשמעה |
| 2 | `digits` | שם המשתנה (אנחנו מקבלים אותו כ-`req.query.digits`) |
| 3 | `,` | מפריד |
| 4 | `1` | מקסימום ספרות |
| 5 | `1` | מינימום ספרות |
| 6 | `8` | timeout בשניות (אחרי כמה שניות ללא הקשה ימות שולחים `digits` ריק) |
| 7 | `NO` | בדיקת תקינות |
| 8 | `no` | ביטול |
| 9 | `yes` | allow_empty (אפשר `digits` ריק בסיום timeout) |

**דוגמאות:**
```
# הקשה אחת, 8 שניות timeout:
read=f-/tts_cache/ab/hash123=digits,,1,1,8,NO,no,yes

# הקשה של 4 ספרות (PIN):
read=f-/tts_cache/ab/hash456=digits,,4,4,15,NO,no,no
```

### 6.3 ניתוק

```
go_to_folder=hangup
```

### 6.4 prefix f- ו-t-

| Prefix | שימוש | דוגמה |
|--------|-------|-------|
| `f-` | קובץ שהועלה לימות | `f-/tts_cache/2a/hash` |
| `t-` | TTS מובנה של ימות (איכות נמוכה) | `t-שלום עולם` |

> ⚠️ **נתיב אחרי `f-` חייב להתחיל ב-`/`!**  
> ימות דוחים נתיבים ללא סלאש מוביל.
> תמיד: `f-${path.startsWith('/') ? path : '/' + path}`

## 7. העלאת קבצי אודיו לימות

### 7.1 Pipeline מלא — Google TTS → ימות

```
טקסט נכנס
    │
    ▼ 1. hash = MD5("he-IL-Wavenet-B:טקסט")[:12]
    │
    ▼ 2. בדיקת אינדקס מקומי (tts-index.json)
    │       ┌─ HIT → מחזיר f-/path מהאינדקס (מיידי!)
    │       └─ MISS ↓
    ▼ 3. שליחה ל-Google Cloud TTS → MP3 זמני
    │
    ▼ 4. UploadFile → ימות (convertAudio=1)
    │
    ▼ 5. שמירה באינדקס JSON + מחיקת MP3 זמני
    │
    ▼ 6. מחזיר f-/tts_cache/xx/hash
    │
    Fallback: t-טקסט (אם Google לא מגיב)
```

### 7.2 Hash — כיצד מחשבים

```js
const crypto = require('crypto');
const hash = crypto
    .createHash('md5')
    .update(`${VOICE_NAME}:${text}`)  // כולל שם קול! שינוי קול = cache miss
    .digest('hex')
    .slice(0, 12);                     // 12 תווים ראשונים
```

**למה כוללים שם קול?** כדי שאם נחליף קול (מ-Wavenet-B ל-Wavenet-D), כל הקבצים ייצרו מחדש.

### 7.3 פיזור תיקיות — למה חשוב

ימות מגבילים **~3,000 קבצים לתיקייה**.  
עם hash של 12 תווים, 2 התווים הראשונים = 256 תיקיות אפשריות.

```js
const subdir = hash.slice(0, 2);          // "2a", "f3", "00", "ff", ...
const yemotPath = `/tts_cache/${subdir}/${hash}`;
const uploadPath = `ivr2:/tts_cache/${subdir}/${hash}.wav`;
```

כך: 256 תיקיות × 3,000 קבצים = **768,000 קבצים אפשריים** — מספיק לכל גודל סביר.

### 7.4 אינדקס מקומי — tts-index.json

```json
{
  "2af3c1hash123": {
    "yemotPath": "/tts_cache/2a/2af3c1hash123",
    "text": "שלום, ברוך הבא למערכת הפנקס",
    "voice": "he-IL-Wavenet-B",
    "createdAt": "2026-04-01T10:00:00.000Z"
  }
}
```

**למה זה קריטי?** בלי אינדקס, כל קריאה לשיחה תצטרך להעלות לגוגל ולימות מחדש — איטי ויקר.

### 7.5 טעינה מקדימה (Preload) — בזמן הפעלת השרת

כל הטקסטים הסטטיים (תפריטים, הנחיות) נוצרים מראש עם עלית השרת:

```js
// server.js
const { getAllStaticPhrases } = require('./ivr/static-phrases');
const { preloadStaticPhrases } = require('./ivr/tts');

// בזמן startup:
await preloadStaticPhrases(getAllStaticPhrases());
console.log('[IVR] Static phrases preloaded');
```

**למה?** ללא preload, השיחה הראשונה תממתין לגוגל TTS — עיכוב של 2-5 שניות בתחילת כל שיחה.

## 8. ניתוב IVR — Routing V2

### 8.1 מה זה Routing V2

**Routing V2** הוא המודל שבחרנו. היתרונות:
- **חינם לחלוטין** (שימוש עצמי)
- **הכי גמיש** — כל לוגיקת השיחה שלנו
- השרת שלנו שולט בכל שלב

**חלופות שלא בחרנו:**
- **Routing V1**: עץ IVR בנוי בלוח הניהול של ימות (פחות גמיש)
- **TTS של ימות** (t-): יקר ואיכות נמוכה — **לא משתמשים בו**

### 8.2 הגדרת הניתוב בלוח ימות

1. כנס לניהול → **"שלוחות"** → **"הגדרות שלוחה"**
2. בחר **Routing V2 / External API**
3. הזן Webhook URL:
   ```
   https://pinkas.cloud/ivr/call?token=hapinkas-ivr-secret-2026
   ```
4. הגדר **Timeout**: 20 שניות מקסימום (מומלץ: שרת יענה תוך 3-5 שניות)
5. **Fallback**: קובץ מוקלט מראש אם השרת לא עונה

## 10. ניהול Session בשיחה

### 10.1 הבעיה — ימות הם Stateless

ימות שולחים **בקשה חדשה לכל אינטראקציה** — אין HTTP session. כל הקשה של המשתמש = GET חדש. צריך לשמור state בין הבקשות.

### 10.2 מה משמש כ-Session ID

```js
const enterId = req.query.ApiYFCallId || req.query.ApiCallId;
```

זה **קבוע לאורך כל השיחה** — זה ה-Session ID שלנו.

### 10.3 מבנה טבלת ivr_sessions

```sql
CREATE TABLE ivr_sessions (
    id          SERIAL PRIMARY KEY,
    user_id     INTEGER UNIQUE,          -- UNIQUE! שיחה חדשה מאפסת session
    enter_id    VARCHAR(100),            -- ApiYFCallId
    phone       VARCHAR(20),
    state       VARCHAR(50) DEFAULT 'init',
    data        JSONB DEFAULT '{}',      -- כל ה-state הדינמי
    created_at  TIMESTAMP DEFAULT NOW(),
    updated_at  TIMESTAMP
);
```

> **`user_id UNIQUE`** — שיחה חדשה של אותו משתמש מחליפה את ה-session הישן (ON CONFLICT DO UPDATE). פשוט ועובד.

### 10.4 State Machine — כל המצבים

```
init
  → waiting_pin          ← אם יש PIN מוגדר
  → menu                 ← תפריט ראשי
      → matches          ← הצעות חדשות
      → all_matches      ← כל ההצעות (כולל ישנות)
      → requests         ← פניות נכנסות ממתינות
      → my_sent          ← פניות שיצאו ממני
      → photos           ← ניהול תמונות
      → messages         ← הודעות חשובות
      → settings         ← שינוי PIN
          → settings_new_pin
          → settings_confirm_pin
```

### 10.5 מבנה data בSession

```js
session.data = {
    state: 'matches',
    items: [...],      // כל ההצעות שנשלפו מה-DB
    index: 0,          // איפה אנחנו ברשימה
    page: 0,           // page offset לשאילתת DB הבאה
    subState: 'l1',    // l1=שכבה בסיסית / l2=פרטים / l3=תיאור מלא
    pinAttempts: 0     // ספירת ניסיונות PIN שגויים
}
```

## 11. אבטחה ו-PIN

### 11.1 אחסון PIN

```js
const bcrypt = require('bcrypt');
const hash = await bcrypt.hash(pin, 10);  // rounds=10
// שמור ב-users.ivr_pin
```

### 11.2 אימות Token

```js
// ב-.env:
IVR_SERVICE_TOKEN=hapinkas-ivr-secret-2026

// בקוד — תמיד startsWith, לא ===
// כי ימות לפעמים מוסיפים פרמטרים ישירות לטוקן
function validateToken(token) {
    return (token || '').startsWith(process.env.IVR_SERVICE_TOKEN);
}
```

> ⚠️ **באג ימות חשוב!** ימות לפעמים מוסיפים פרמטרים ישירות לתוך ה-token ב-query string:  
> `?token=hapinkas-ivr-secret-2026?ApiCallId=abc123`  
> לכן חייבים `startsWith` ולא `===`.

### 11.3 חסימה אחרי 3 ניסיונות שגויים

```
checkPin(userId, inputPin):
  ┌─ blocked_until > NOW → return 'blocked' (חסום עוד X דקות)
  ├─ אין PIN → return 'no_pin' (לא הגדיר PIN)
  ├─ bcrypt.compare match → reset attempts → return 'ok'
  └─ no match:
      attempts++
      if attempts >= 3 → blocked_until = NOW + 30min → return 'blocked'
      else → return 'wrong' (נשארו X ניסיונות)
```

### 11.4 עמודות IVR בטבלת users

```sql
ALTER TABLE users ADD COLUMN IF NOT EXISTS ivr_pin VARCHAR(255);
ALTER TABLE users ADD COLUMN IF NOT EXISTS allow_ivr_no_pass BOOLEAN DEFAULT FALSE;
ALTER TABLE users ADD COLUMN IF NOT EXISTS ivr_failed_attempts INTEGER DEFAULT 0;
ALTER TABLE users ADD COLUMN IF NOT EXISTS ivr_blocked_until TIMESTAMP;
```

### 13.3 מבנה router.js — Skeleton

```js
const express = require('express');
const router = express.Router();
const { validateToken, getUserByPhone, checkPin, getSession, saveSession } = require('./auth');
const { textToUrl } = require('./tts');

router.get('/call', async (req, res) => {
    const { token, ApiPhone, digits } = req.query;
    
    // 1. אימות token
    if (!validateToken(token)) {
        return res.status(403).send('');
    }
    
    // 2. ניתוק
    if (req.query.hangup === 'yes') {
        // cleanup session
        return res.type('text').send('');
    }
    
    // 3. קבלת session
    const enterId = req.query.ApiYFCallId || req.query.ApiCallId;
    let session = await getSession(enterId, ApiPhone);
    
    // 4. נרמול digits
    const key = digits != null ? String(digits).trim() || null : null;
    
    // 5. State Machine
    let response;
    switch (session.state) {
        case 'init':     response = await handleInit(session, req); break;
        case 'waiting_pin': response = await handlePin(session, key); break;
        case 'menu':     response = await handleMenu(session, key); break;
        case 'matches':  response = await handleMatches(session, key); break;
        // ... שאר המצבים
    }
    
    // 6. שמירת session ושליחת תגובה
    await saveSession(session);
    res.type('text').send(response);
});

module.exports = router;
```

> **גילוי חשוב:** הספרייה `yemot-router2` היא **הדרך הנכונה לעבוד עם Routing V2**. ניסיון לבנות state machine ידני עם Express Router גרידא הוא כאוב ומועד לכשלון. הספרייה מטפלת בניהול session, פורמט תגובות, ו-ExitError אוטומטית.

### 18.1 התקנה

```bash
npm install yemot-router2
```

### 18.2 מבנה בסיסי

```js
const { YemotRouter, ExitError } = require('yemot-router2');

const router = YemotRouter({
    printLog: true,           // לוגים לדיבאג (כבה ב-production)
    timeout: 5 * 60 * 1000,  // timeout של 5 דקות לשיחה
    defaults: { removeInvalidChars: true },
    uncaughtErrorHandler: async (err, call) => {
        if (err instanceof ExitError) return;  // חובה! ExitError אינו שגיאה
        console.error('IVR Error:', err.message);
        try {
            call.id_list_message([{ type: 'text', data: 'אירעה שגיאה', removeInvalidChars: true }]);
        } catch (e) { /* ExitError or call ended */ }
    }
});

router.get('/call', async (call) => {
    // השיחה מטופלת כאן — call הוא אובייקט עם כל הפרמטרים
    const phone = call.ApiPhone;
    const did   = call.ApiDID || call.ApiRealDID;
    // ...
});

module.exports = router;
```

### 18.4 call.read() — השמעה + קבלת קלט

```js
// recPath = נתיב הקובץ המוקלט בשרתי ימות (ללא ivr2:)

const transcript = await call.read(

```

### 18.5 סוגי message segments

```js
// קובץ שהועלה לימות:
{ type: 'file', data: '/tts_cache/2a/hash123' }
// שים לב: data הוא הנתיב ב-ivr2 — ללא 'ivr2:' וללא 'f-'

// טקסט TTS של ימות (איכות נמוכה, חינם):
{ type: 'text', data: 'שלום עולם', removeInvalidChars: true }
```

**helper נוח לשימוש עם Google TTS:**
```js
function toYRMessage(segment) {
    if (segment.startsWith('f-')) {
        return { type: 'file', data: segment.slice(2) };
    }
    const text = segment.startsWith('t-') ? segment.slice(2) : segment;
    return { type: 'text', data: text, removeInvalidChars: true };
}
```

### 18.6 call.id_list_message() — השמעה ללא קלט

```js
// השמעה ויציאה מהפונקציה:
call.id_list_message([{ type: 'text', data: 'תודה להתראות', removeInvalidChars: true }]);
// זורק ExitError אוטומטית — הקוד שאחרי לא יורץ!

// השמעה לפני פעולה (prependToNextAction):
call.id_list_message([{ type: 'file', data: '/path/to/audio' }], { prependToNextAction: true });
call.hangup();  // יישמע הקובץ ואז ינתק
```

### 18.7 call.hangup() — ניתוק

```js
call.hangup();  // ניתוק מיידי
```

### 18.8 ExitError — הבנה קריטית

`ExitError` **אינו שגיאה!** הוא mechanism של הספרייה לסיים את הפונקציה הנוכחית.

- `call.id_list_message()` ו-`call.hangup()` **זורקים ExitError** לאחר הביצוע
- ה-`uncaughtErrorHandler` חייב לדלג עליו: `if (err instanceof ExitError) return;`
- בתוך try/catch — חייב `if (err instanceof ExitError) throw err;` (לא לבלוע!)

```js
try {
    nameRecPath = await call.read([...], 'record', {...});
} catch (err) {
    if (err instanceof ExitError) throw err;  // ⚠️ חובה!
    console.warn('Recording failed:', err.message);
    callerHungUp = true;
}
```

### 18.9 Pipeline אסינכרוני אחרי השיחה

חשוב: הפעל עיבוד לאחר שיחה **לפני** `id_list_message` (שזורק ExitError):

```js
// ✅ נכון — pipeline מתחיל לפני הניתוק
(async () => {
    try {
        await processCallAsync(pipelineData);
    } catch (err) {
        console.error('pipeline error:', err.message);
    }
})();

// עכשיו נסיים את השיחה
call.id_list_message([toYRMessage(confirmSeg)], { prependToNextAction: true });
call.hangup();

// ❌ שגוי — הפעלה אחרי id_list_message לא תגיע לעולם
await processCallAsync(pipelineData);  // קוד מת!
```

## 19. הקלטת שיחות — Record Mode

### 19.1 איך Record Mode עובד

בשיטת `record`:
- ימות מקליטים את הקול של המתקשר
- הקלטה מסתיימת כאשר: המתקשר לוחץ `#` **או** מנתק
- `save_on_hangup: true` — שומר גם אם ניתק ללא `#`
- הנתיב לקובץ מוחזר מ-`call.read()` (למשל `/1234567890.wav`)

```js
const recPath = await call.read(
    [toYRMessage(greetingSeg)],
    'record',
    {
        no_confirm_menu: true,   // ללא תפריט "לשמירה לחץ 1"
        save_on_hangup: true,    // שמור גם אם ניתק בלי #
        max_length: '60'         // מקסימום 60 שניות (10 לשם, 60 להודעה)
    }
);
// recPath = '/1234567890.wav' (ללא ivr2: בהתחלה)
```

### 19.2 הנחיות חשובות למשתמש

חייבים להגיד למתקשר **בפירוש** איך לסיים:

```js
const fullGreeting = `${greeting} אנא אמרו את שמכם ובסיום הקישו סולמית`;
const askMsgText   = `מה תרצה להעביר לX בסיום הקישו סולמית`;
```

> הציבור הישראלי רגיל ללחוץ `#` בסיום הקלטה (בנקים, מוקדים). זה טבעי.

### 19.3 Yemot STT (חלופה — תמלול מובנה)

ימות מספקים תמלול מובנה (STT) **ללא עלות נוספת**:

```js
const transcript = await call.read(
    [messages],
    'stt',
    {
        use_records_recognition_engine: true,
        quiet_max: '2',     // סיום אוטומטי אחרי 2 שניות שקט
        length_max: '60'    // מקסימום שניות
    }
);
```

**יתרון:** זיהוי שקט אוטומטי — המשתמש לא צריך ללחוץ `#`  
**חיסרון:** איכות נמוכה יותר מ-Google STT, במיוחד לשמות פרטיים ומספרים

**ניתן לשלב** — לשמור גם קובץ וגם לקבל תמלול:
```js
// file_name = שם הקובץ שישמר (ימות ישמרו גם הקלטה וגם יתמללו)
const transcript = await call.read([msgs], 'stt', {
    use_records_recognition_engine: true,
    quiet_max: '2',
    length_max: '60',
    file_name: 'message_recording'  // ימות ישמרו קובץ בשם זה
});
```

### 19.4 השוואה: Record vs STT

| קריטריון | Record + Google STT | Yemot STT |
|----------|---------------------|-----------|
| **דיוק עברית** | ⭐⭐⭐⭐⭐ מצוין | ⭐⭐⭐ בינוני |
| **שמות פרטיים** | ⭐⭐⭐⭐ טוב | ⭐⭐ חלש |
| **מספרי טלפון** | ⭐⭐⭐⭐ טוב | ⭐⭐ חלש |
| **זיהוי שקט** | ❌ ידני (`#`) | ✅ אוטומטי |
| **עלות** | חינם (מכסת גוגל) | חינם (ימות) |
| **המלצה** | ✅ לשימוש ב-Production | טסטים / MVP מהיר |

> **מסקנה:** לשירות מקצועי — תמיד Google STT. "שגיאה קטנה בתמלול = אובדן לקוח".

## 20. הורדת הקלטות מימות

### 20.1 נתיב הקובץ — נרמול

`call.read()` מחזיר נתיב **ללא** `ivr2:` (למשל `/1234567890.wav`).  
כדי להוריד — צריך להוסיף `ivr2:`:

```js
const normalizedPath = yemotPath.startsWith('ivr2:')
    ? yemotPath
    : `ivr2:${yemotPath.startsWith('/') ? yemotPath : '/' + yemotPath}`;
```

### 20.2 הורדה — Endpoint ומבנה

```js
const response = await axios.get(`${YEMOT_API_BASE}/DownloadFile`, {
    params: {
        token,               // Token מ-Login
        path: normalizedPath // ivr2:/1234567890.wav
    },
    responseType: 'arraybuffer',  // חובה! לא 'json'
    timeout: 30000
});

const buf = Buffer.from(response.data);
fs.writeFileSync(localPath, buf);
```

> ⚠️ **חובה:** `responseType: 'arraybuffer'` — בלי זה הנתונים יקולקלו.  
> ⚠️ **בדוק Content-Type:** אם ימות מחזירים שגיאה, הם שולחים JSON עם content-type `application/json`. תמיד בדוק:

```js
const contentType = response.headers['content-type'] || '';
if (contentType.includes('json') || contentType.includes('text')) {
    const errText = buf.toString('utf8').slice(0, 200);
    throw new Error(`Yemot error: ${errText}`);
}
```

### 20.3 Race Condition — בעיה קריטית!

**הבעיה:** הפייפליין מתחיל מיד אחרי שהמתקשר לוחץ `#` — אבל ימות עדיין לא סיימו לשמור את הקובץ.  
**תוצאה:** הורדה של קובץ ריק (388 bytes = header בלבד).

**הפתרון:**
```js
// המתן לפני הורדה ראשונה
await sleep(4000);  // 4 שניות

await downloadRecording(messageRecPath, localRecPath);
const size = fs.statSync(localRecPath).size;

// אם עדיין קטן — נסה שוב
if (size < 5000) {
    await sleep(4000);  // עוד 4 שניות
    await downloadRecording(messageRecPath, localRecPath);
}
```

### 20.4 אבחון — קבצים חשודים

שמור קבצים קטנים לבדיקה ידנית במקום למחוק:

```js
const stats = fs.statSync(localRecPath);
if (stats.size < 5000 || !transcript) {
    const debugPath = localRecPath.replace('/temp/', '/temp/debug_');
    fs.renameSync(localRecPath, debugPath);
    // בדוק ידנית — play ב-Audacity לדעת מה בתוכו
} else {
    fs.unlink(localRecPath, () => {});
}
```

## 21. Google STT לעברית עם קבצי ימות

### 21.1 פרמטרים שעובדים לעברית

```js
const [response] = await speechClient.recognize({
    config: {
        languageCode: 'iw-IL',   // עברית — 'iw-IL' ולא 'he-IL'!
        // ⚠️ אל תגדיר model — ברירת המחדל היא הטובה ביותר לעברית
        // ⚠️ אל תגדיר useEnhanced — לא נתמך לעברית
        // ⚠️ אל תגדיר alternativeLanguageCodes — לא נתמך במודל ברירת מחדל
    },
    audio: { content: audioContent }
});
```

**שגיאות שיצאו לנו ופתרוניהן:**

| שגיאה | סיבה | פתרון |
|-------|------|--------|
| `phone_call model is currently not supported for language: iw-IL` | `model: 'phone_call'` | הסר `model` לחלוטין |
| `latest_long not supported` | `model: 'latest_long'` | הסר `model` לחלוטין |
| `alternative_language_codes currently not supported` | `alternativeLanguageCodes` עם מודל לא תואם | הסר `alternativeLanguageCodes` |

### 21.2 קבצי WAV מימות — בעיית Encoding

קבצי WAV מימות הם **8kHz** ויכולים להיות בכמה encodings שונים.  
הפתרון — נסה כמה אסטרטגיות לפי הסדר:

```js
const STRATEGIES = [
    { name: 'auto-detect',   config: { sampleRateHertz: 8000 } },           // ✅ עובד ברוב המקרים
    { name: 'MULAW 8kHz',    config: { encoding: 'MULAW',    sampleRateHertz: 8000 } },
    { name: 'LINEAR16 8kHz', config: { encoding: 'LINEAR16', sampleRateHertz: 8000 } },
    { name: 'LINEAR16 16kHz',config: { encoding: 'LINEAR16', sampleRateHertz: 16000 } },
    { name: 'ALAW 8kHz',     config: { encoding: 'ALAW',     sampleRateHertz: 8000 } }
];

for (const strategy of STRATEGIES) {
    try {
        const [response] = await speechClient.recognize({
            config: { languageCode: 'iw-IL', ...strategy.config },
            audio: { content: audioContent }
        });
        const transcript = response.results
            .map(r => r.alternatives[0]?.transcript || '')
            .join(' ').trim();
        if (transcript) {
            console.log(`STT הצליח עם: ${strategy.name}`);
            return transcript;
        }
    } catch (err) {
        console.warn(`STT נכשל עם ${strategy.name}:`, err.message);
    }
}
```

> **הערה:** `auto-detect` (ללא `encoding`) עובד הכי טוב כי Google קורא את header ה-WAV ומזהה encoding אוטומטית.

## 23. באגים קריטיים שגילינו

### 23.1 Token עם `?` כפול — הבאג החשוב ביותר

**התופעה:** ימות שולחים את ה-Webhook עם ה-token כך:
```
?token=mytoken?ApiCallId=abc123&ApiPhone=054...
```
במקום:
```
?token=mytoken&ApiCallId=abc123&ApiPhone=054...
```

**תוצאה:** `yemot-router2` לא מזהה את הבקשה ("the request is not valid yemot request") כי הטוקן "מגיע" עם `?ApiCallId=...` בתוכו.

**הפתרון — middleware חובה:**
```js
function yemotTokenFix(req, res, next) {
    if (req.query.token && req.query.token.includes('?')) {
        const parts = req.query.token.split('?');
        req.query.token = parts[0];  // רק הטוקן האמיתי
        for (let i = 1; i < parts.length; i++) {
            const pairs = parts[i].split('&');
            for (const pair of pairs) {
                const [k, v] = pair.split('=');
                if (k && !req.query[k]) req.query[k] = decodeURIComponent(v || '');
            }
        }
    }
    next();
}

// שימוש — חובה לפני YemotRouter!
app.use('/vs/call', yemotTokenFix);
app.use('/vs', vsRouter);
```

### 23.2 record= לא עובד ב-Routing V2

**שגיאה:** שליחת `record=` כתגובת טקסט לימות לא עובדת ב-Routing V2.  
**פתרון:** להשתמש רק דרך `yemot-router2` עם `call.read([...], 'record', {...})`.

### 23.3 convertAudio=1 — חובה בהעלאת MP3

**שגיאה:** העלאת MP3 ללא `convertAudio=1` → הקובץ לא נשמע (שקט מוחלט).  
**פתרון:** תמיד `form.append('convertAudio', '1')` בהעלאה.

### 23.4 נתיב f- חייב להתחיל ב-/

**שגיאה:** `f-tts_cache/2a/hash` → שקט.  
**פתרון:** `f-/tts_cache/2a/hash` (סלאש מוביל חובה).

### 23.5 ניסיון להשתמש ב-Did_Go_To.ini עם go_to_folder

**שגיאה:** הגדרת `type=go_to_folder` בשלוחה הראשית גורמת להתעלמות מ-`check_did_and_go_to_folder`.  
**פתרון:** חשבון ימות נפרד לכל מערכת (ראה סעיף 22).

### 23.6 Race Condition בהורדת הקלטה

**שגיאה:** הקובץ שמוריד הוא 388 bytes (header בלבד) — ימות עדיין לא סיימו לשמור.  
**פתרון:** `await sleep(4000)` לפני הורדה, ו-retry אם קטן מ-5KB (ראה סעיף 20.3).

### 23.7 Google STT לא תומך ב-`phone_call` model לעברית

**שגיאה:** `Invalid recognition 'config': The phone_call model is currently not supported for language: iw-IL`  
**פתרון:** הסר `model` לחלוטין — השתמש בברירת מחדל של Google (ראה סעיף 21).
