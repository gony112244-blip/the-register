# מערכת הטלפון (IVR) — התקדמות, גיבוי בגיט, ומה הלאה

**עודכן:** אפריל 2026  
**ספק:** ימות המשיח · **TTS:** Google Cloud · **אפליקציה:** Node.js + PostgreSQL

---

## 1. גיבוי בגיט — מצב נוכחי

- **הקוד המקורי** (לא כולל קבצי ריצה על השרת) אמור לחיות ב־GitHub בענף `main`.
- בתיקיית הפיתוח המקומית: אם `git status` מראה *working tree clean* ו־*up to date with origin/main* — **הכל שמור בענן**.
- **לא נכנס לגיט בכוונה** (מוגדר ב־`.gitignore`):
  - `node_modules/`
  - `.env`, קבצי credentials
  - **`ivr-audio/`** — קבצי MP3, אינדקס TTS מקומי, תיקיות `static` / `dynamic` / `temp` (נוצרים בשרת בזמן ריצה)

**קומיטים אחרונים רלוונטיים (דוגמה):**

| קומיט | נושא |
|--------|------|
| `chore: exclude ivr-audio from git` | מניעת `git add -A` שמכניס אודיו לריפו |
| `fix: digits trim safety + leading slash for yemot paths` | ייצוב פרמטר `digits` + נתיבי `f-` לימות |

**חשוב לתפעול:** לא לבצע בשרת `git add -A && git commit` בלי `git status` — קל לכלול בטעות `node_modules` ו־`ivr-audio`. אם השרת «התפצל» מ־GitHub, הפתרון הנכון הוא `git fetch origin && git reset --hard origin/main` (אחרי גיבוי `.env`), לא מיזוג של קומיט ענק מקומי.

## 2. מה כבר מיושם במערכת הטלפונית

### תשתית וקבצים מרכזיים

| רכיב | קבצים / תפקיד |
|------|----------------|
| אימות ומשתמש | `ivr/auth.js` — token ימות, זיהוי לפי מספר, PIN (bcrypt), חסימה זמנית |
| נתונים | `ivr/data.js` — שאילתות ייעודיות ל־IVR |
| זרימת שיחה | `ivr/router.js` — state machine, תפריטים, מקשים |
| TTS | `ivr/tts.js` — Google TTS, מטמון מקומי לפי hash, החזרת נתיבי השמעה לימות (`f-...`) |
| API ימות | `ivr/yemotApi.js` — Login, העלאת קבצים, מחיקה, בדיקת קיום |
| ביטוי קבוע | `ivr/static-phrases.js` + טעינה מקדימה (ב־`server.js`) |
| אודיו פרופיל | `ivr/profileAudio.js` — L1/L2/L3, קריאה משמירת/עדכון פרופיל באתר |
| ניקוי | `ivr/cleanupJob.js` — מחיקה תקופתית של אודיו פרופילים ישנים בימות (לפי `tts_last_played`) |
| הגייה | `ivr/phonetic-map.json` |

### התנהגות עסקית (ברמה גבוהה)

- כניסה לפי Caller ID, PIN (או דילוג אם `allow_ivr_no_pass`), מבזק סטטוס בהתחלה.
- תפריט ראשי: הצעות, פניות נכנסות, פניות שיצאו, ניהול תמונות, הודעות, שינוי PIN, תמיכה, חזרה עם `#`.
- שלוש שכבות פרטים לפרופיל (מקשים 4 ו־5 במקומות הרלוונטיים).
- התאמת ניסוח לפי מגדר; pagination בשיחה.

### שינויי DB (סכימה)

- `users`: עמודות IVR (PIN, ניסיונות, חסימה, `allow_ivr_no_pass`, וכו׳) + שדות הקשורים ל־TTS פרופיל לפי הצורך.
- `ivr_sessions`, טבלאות עזר (`ivr_logs`, `ivr_match_views` וכו׳) לפי מה שמוגדר ב־`server.js`.

## 3. מה עוד צריך / רצוי להתקדם

### לפני / סביב השקה מלאה

| משימה | הערות |
|--------|--------|
| **Webhook ימות** | כתובת יציבה (HTTPS) ל־`/ivr/call` עם `token` סודי כפי שבסביבה |
| **בדיקות שיחה אמיתיות** | מעבר על כל המקשים עם משתמש אמיתי (כולל קצוות: אין ספרות, ניתוק, PIN שגוי) |
| **ניקוי אודיו לפני שינוי קול** | אם מחליפים קול/מודל TTS: לרוקן מטמון רלוונטי בשרת (`ivr-audio`) כדי שלא יושמעו קבצים ישנים |
| **סנכרון שרת עם Git** | רק `pull` אחרי קומיטים נקיים מהמחשב המקומי; ללא קומיטים מ־`/var/www/...` עם `add -A` |

### שיפורים לשלב הבא (עדיפות בינונית–נמוכה)

| נושא | תיאור קצר |
|------|-----------|
| סימון «נשמע בטלפון» באתר | שימוש ב־`ivr_match_views` (או מנגנון דומה) ליד הצעות |
| מיילים כמו באתר | אירועים שמגיעים מהאתר — לוודא פריטי IVR מפעילים אותם אם נדרש |
| ביטול חיבור *active* מהטלפון | היום ייתכן שרק ביטול *pending* מלא |
| לוג שיחות מפורט | משך, יציאה לפי מקש |
| Timeout והקראה חוזרת | אחרי שקט ארוך |
| בקשת תמונה מהטלפון | לצד עם חיבור פעיל |
