# מערכת צ'ינטוקים (Tzintuk)

שיחות התראה קצרות ("מיסקול") דרך ימות המשיח, מהמספר השלישי, כדי להזכיר למשתמשים שיש להם משהו חדש לראות.
המערכת **פרוסה אך כבויה לחלוטין** (`TZINTUK_ENABLED=false`) ולא מחייגת עד אישור ידני.

---

## מתי נשלח צ'ינטוק?

שני סוגי אירועים:

1. **התאמה חדשה (`new_match`)** — פעם ראשונה בלבד. גם אם הצעה ירדה מבירורים וחזרה — לא יישלח שוב.
   מזוהה ע"י job שרץ כל 15 דקות (`scanNewMatches`).
2. **בקשת התקדמות** ממשתמש אחר:
   - `interest` — מישהו שלח פנייה (`/connect`).
   - `photo_request` — מישהו ביקש לראות תמונות (`/request-photo-access`).
   - `reference_request` — מישהו ביקש ממליץ נוסף (`/request-additional-reference`).
   - `shadchan` — שני הצדדים אישרו והתיק עבר לשדכנית (`/finalize-connection`).

## חיסכון בצ'ינטוקים — שליחה בערב בלבד

האירועים **לא מחייגים מיד**. הם נצברים במהלך היום (סטטוס `pending`).
פעם ביום, **בזמן אקראי בין 19:00 ל-21:00 (שעון ישראל)**, רץ `dispatch`:

- משתמש ש**כבר ראה** את ההודעה (התחבר לאתר אחרי שהאירוע נוצר) — **מדלגים** (`skipped_seen`).
- משתמש ש**עדיין לא ראה** — מקבל **צ'ינטוק אחד מאוחד** (כל שאר ההתראות שלו מאותו יום מסומנות `batched`).

> שעון: השרת רץ ב-UTC, לכן החלון מחושב במפורש לפי `Asia/Jerusalem` (לא לפי שעון השרת).
> "ראה" = `users.last_login` מאוחר מ-`created_at` של האירוע.

## הגדרות (`.env` בשרת `/var/www/hapinkas/.env`)

| משתנה | ערך נוכחי | משמעות |
|---|---|---|
| `TZINTUK_ENABLED` | `false` | מתג ראשי. כבוי = לא מחייג כלל. |
| `TZINTUK_DRY_RUN` | `true` | גם אם ENABLED=true — רק רושם בלוג, לא מחייג. |
| `TZINTUK_CALLER_ID` | `0775017539` | המספר המחייג (זיהוי יוצא). |
| `TZINTUK_WINDOW_START_HOUR` | `19` | תחילת חלון השליחה (שעון ישראל). |
| `TZINTUK_WINDOW_END_HOUR` | `21` | סוף חלון השליחה (שעון ישראל). |
| `TZINTUK_TZ` | (ברירת מחדל `Asia/Jerusalem`) | אזור הזמן של החלון. |
| `TZINTUK_SCAN_INTERVAL_MS` | (ברירת מחדל `900000`) | תדירות סריקת התאמות (15 דק'). |

**חיוג אמיתי קורה רק כש-`TZINTUK_ENABLED=true` וגם `TZINTUK_DRY_RUN=false`.**

## איך מפעילים (כשתחליט)

1. עריכת `.env` בשרת:
   ```bash
   ssh root@pinkas.cloud
   cd /var/www/hapinkas
   nano .env       # TZINTUK_ENABLED=true  ,  TZINTUK_DRY_RUN=false
   pm2 restart hapinkas --update-env
   ```
2. מומלץ קודם להפעיל עם `TZINTUK_ENABLED=true` + `TZINTUK_DRY_RUN=true` ליום-יומיים ולבדוק את הלוג, ורק אז `DRY_RUN=false`.

### שלב ביניים מומלץ — "seed" של ה-seen-set
ה-job רושם התאמות קיימות גם במצב כבוי/dry-run (סטטוס `skipped_disabled`/`dry_run`).
כך, כשתעבור לחיוג אמיתי — **רק התאמות חדשות מאותו רגע ואילך יחייגו**, ולא כל ההיסטוריה.

## ⚠️ תלות חיצונית חשובה — זיהוי יוצא בימות

בדיקת `GetApprovedCallerIDs` בחשבון מחזירה:

```json
"call": {
  "mainDid": "+972772291821",
  "secondaryDids": ["+97223130778", "+972775017539"],
  "callerIds": ["+972772291821"]
}
```

- `0775017539` (`+972775017539`) **שייך לחשבון** (מופיע ב-`secondaryDids`).
- אך **אינו ברשימת ה-`callerIds` המאושרים** לזיהוי יוצא (שם רק `0772291821`).

**משמעות:** ייתכן שחיוג אמיתי עם `callerId=0775017539` יחזיר שגיאה `120` (זיהוי יוצא לא מאושר).
לפני הפעלת חיוג אמיתי — יש לאמת מול ימות ש-`0775017539` מאושר כזיהוי יוצא
(בפאנל ימות / שירות לקוחות / `ValidationCallerId`), או לשנות את `TZINTUK_CALLER_ID` למספר מאושר.

## בדיקה (אדמין)

- `GET /admin/tzintuk-log?limit=50` — סטטוס + ספירת סטטוסים + הלוג האחרון.
- `POST /admin/tzintuk-test` (גוף JSON):
  - `{ "action": "caller-ids" }` — בדיקה אם `0775017539` מאושר/בבעלות החשבון.
  - `{ "action": "scan" }` — הרצת סריקת התאמות ידנית.
  - `{ "action": "dispatch" }` — הרצת ה-dispatch הערבי ידנית (במצב כבוי/dry-run לא מחייג).
  - `{ "userId": 12, "eventType": "interest", "relatedUserId": 34 }` — תזמון אירוע בודד (pending).

## רכיבים בקוד

| רכיב | קובץ |
|---|---|
| API לימות (`runTzintuk`, `getApprovedCallerIds`) | `ivr/yemotApi.js` |
| לוגיקת השירות (notify / dispatch / scan / תזמון) | `ivr/tzintuk.js` |
| טבלה `tzintuk_log` + עמודת `users.tzintuk_opt_out` | `updateDbSchema()` ב-`server.js` |
| חיבור hooks + נתיבי אדמין + הפעלת scheduler | `server.js` |

### סטטוסים בטבלת `tzintuk_log`
`pending` (ממתין לערב) · `dry_run` · `sent` · `failed` · `batched` (אוחד לשיחה אחת) ·
`skipped_seen` (כבר ראה) · `skipped_duplicate` (כבר נשלח new_match) · `skipped_no_phone` ·
`skipped_disabled` (המערכת כבויה) · `skipped_opt_out`.

### קודי שגיאה מימות (מטופלים)
`102` אין מספרים תקינים · `103` אין מספיק יחידות · `104` שבת/חג (ימות חוסם אוטומטית) · `120` callerId לא מאושר.
