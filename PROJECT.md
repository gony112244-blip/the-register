# הפנקס — מאפיין פרויקט

מסמך זה מסכם את מערכת **הפנקס**: מטרה, ארכיטקטורה, טכנולוגיות ויכולות עיקריות. מתעדכן לפי מצב הקוד בפרויקט.

---

## 1. מהות המוצר

**הפנקס** היא פלטפורמת **שידוכים וזיווג** מבוססת אינטרנט, מיועדת לקהילה הדתית/חרדית. המשתמשים יוצרים פרופיל, מקבלים הצעות התאמה, שולחים **פניות** ובקשות **צפייה בתמונות**, מתקשרים דרך **הודעות** במערכת, וניתן להעביר זוגות לטיפול **שדכנית** (כולל שליחת כרטיסיות במייל). יש **ממשק מנהל** לאישור פרופילים, ניהול משתמשים, תמיכה ומעקב.

---

## 2. ארכיטקטורה כללית

| שכבה | תיאור |
|------|--------|
| **לקוח** | SPA ב-React (Vite), עברית RTL, PWA (התקנה כאפליקציה) |
| **שרת** | Node.js + Express — API REST, אימות JWT, שליחת מיילים |
| **מסד נתונים** | PostgreSQL — משתמשים, קשרים, הודעות, תמונות, לוג פעילות וכו' |
| **פריסה טיפוסית** | שרת לינוקס: `pm2` להרצת `server.js`, קבצים סטטיים מ-`frontend/dist` |

הפרויקט הוא **מונורפו**: שורש הפרויקט = backend, תיקייה `frontend/` = הלקוח.

---

## 3. מחסנית טכנולוגית

### Backend (`package.json` בשורש)
- **Express 5**, **pg** (PostgreSQL), **bcrypt**, **jsonwebtoken**
- **nodemailer** — מיילים (ייצור: Ethereal; פרודקשן: SMTP לפי הגדרה)
- **helmet**, **cors**, **express-rate-limit** — אבטחה בסיסית
- **multer** — העלאת קבצים
- **dotenv** — משתני סביבה

### Frontend (`frontend/package.json`)
- **React 19**, **react-router-dom 7**
- **Vite** (rolldown-vite), **vite-plugin-pwa**
- **Chart.js** — דשבורד מנהל

## 5. יכולות משתמש (תמצית)

- **הרשמה והתחברות** — מזהה עיקרי: טלפון; אימות מייל אופציונלי
- **פרופיל עשיר** — שדות דמוגרפיים, משפחה, לימודים, העדפות חיפוש, תמונות
- **אישור פרופיל** — עד שהמנהל מאשר, גלישה מוגבלת (לפי הלוגיקה בפרונט/שרת)
- **התאמות (Matches)** — רשימת מועמדים לפי מגדר מנוגד וקריטריוני חיפוש
- **פניות לפרטי התקשרות** — `connections`: שליחה, אישור, דחייה, ביטול; שלבים כולל מעבר לשדכנית
- **בקשות תמונות** — `photo_approvals`: אישור/דחייה/אישור תמיד
- **הודעות** — תיבת דואר פנימית + התראות
- **הסתרת פרופילים** — "סל מיחזור" עם סיבה
- **חסימת משתמשים** — מניעת פניות חוזרות / הטרדה
- **מחיקת חשבון** — מהפרופיל
- **יצירת קשר** — טופס פניות תמיכה (`/contact`) + מעקב למנהל
- **PWA** — התקנה מהדפדפן

---

## 6. יכולות מנהל (תמצית)

- דשבורד, סטטיסטיקות, אישור/דחיית פרופילים ממתינים
- ניהול משתמשים: צפייה בפרופיל מלא, הערות פנימיות, שליחת הודעת מערכת, חסימה/מחיקה
- **לוג פעילות** (`activity_log`) — ציר זמן של אירועים (פניות, תמונות, שדכנית, הסתרות, חסימות וכו') בלשונית היסטוריה בניהול משתמשים
- **פניות תמיכה** — `support_tickets` / תשובות, מייל + הודעה במערכת
- שידוכים ושליחת כרטיסיות לשדכנית, סגירת שידוכים וכו' (לפי הנתיבים ב-`server.js`)

---

## 7. מסד נתונים

- **PostgreSQL**; חיבור דרך `pg` Pool
- סכמה מתעדכנת בזמן ריצה בפונקציה **`updateDbSchema()`** ב-`server.js` (טבלאות ועמודות חדשות בלי מיגרציות נפרדות)
- טבלאות לדוגמה: `users`, `connections`, `messages`, `photo_approvals`, `hidden_profiles`, `notifications`, `user_blocks`, `support_tickets`, `support_replies`, `activity_log`, ועוד

---

## 8. אבטחה ואימות

- **JWT** בכותרת `Authorization: Bearer`
- סיסמאות מגובות ב-**bcrypt**
- מגבלות קצב (rate limit) על חלק מהנתיבים
- **helmet** לכותרות HTTP

---

## 9. מיילים

- תבניות HTML (בשרת / קבצי תבניות) — אימות, ברוכים הבאים, התראות על פעולות, תמיכה וכו'
- הגדרה דרך משתני סביבה: שירות SMTP, משתמש/סיסמה, כתובת שולח

---

## 10. הרצה מקומית (סיכום)

1. PostgreSQL זמין ומסד נתונים מוגדר (ראו `setup_database.js` / `.env`).
2. שורש: `npm install` → `node server.js` (או `npm start`).
3. `frontend/`: `npm install` → `npm run dev` (פיתוח) או `npm run build` (ייצור).

בפרודקשן: ה-API משרת גם את ה-`dist` של הפרונטאנד (לפי ההגדרה ב-`server.js`).

---

## 11. פריסה לשרת (תזכורת)

```bash
cd /var/www/hapinkas   # או נתיב הפרויקט אצלכם
git pull origin main
cd frontend && npm run build && cd ..
pm2 restart hapinkas
```

לאחר עדכון UI — לעיתים נדרש רענון חזק / אינקוגניטו בגלל **PWA cache**.

## 12. משתני סביבה נפוצים (לא מלא)

| משתנה | תפקיד |
|--------|--------|
| `PORT` | פורט השרת |
| `JWT_SECRET` | חתימת JWT |
| `DATABASE_URL` או פרמטרי `pg` | חיבור PostgreSQL |
| `EMAIL_USER`, `EMAIL_PASS`, `EMAIL_SERVICE` | שליחת מייל |
| `APP_URL` | קישורים במיילים |
| `NODE_ENV` | `production` וכו' |

פרונטאנד: `VITE_API_URL` — בפרודקשן לעיתים ריק כשה-API באותו דומיין.
