# 📋 סיכום פרויקט: "הפנקס" (The Register)
## מערכת ניהול ושידוכים מתקדמת

---

## 1. ארכיטקטורה ומבנה נתונים (Schema)

המערכת מבוססת על מסד נתונים רלציוני (PostgreSQL) עם דגש על הפרדה בין נתוני משתמש גלויים, נתונים אישיים מאובטחים, וניהול קשרים רב-שלבי.

### תרשים קשרים (ERD) - עיקרי
- **users (משתמשים):** הישות המרכזית. מכילה פרופיל מלא, העדפות חיפוש, סטטוס אישור וניהול שינויים.
- **connections (קשרים):** מנהלת את ה"שידוכים" בין גולשים. סטטוסים: `pending` -> `active` -> `waiting_for_shadchan` -> `successful`/`rejected`.
- **photo_approvals (אישורי תמונות):** מנגנון נפרד לצפייה בתמונות. אישור הוא הדדי ומוגבל בזמן (48 שעות).
- **messages (הודעות):** מערכת דיוור פנימית (התראות מערכת, הודעות מנהל, עדכוני סטטוס).
- **shadchaniot (שדכניות):** ישות ניהולית שאליה משויכים תיקים שהגיעו לשלב המתקדם.
- **user_images (תמונות):** שמירת נתיבי תמונות פרופיל (עד 3 למשתמש).

### מפרט שדות עיקרי (טבלת `users`)
| שדה | סוג | הגבלות/ערכי ברירת מחדל | תיאור |
|:---|:---|:---|:---|
| `id` | Integer | Primary Key, Auto-increment | מזהה ייחודי |
| `phone` | Varchar(20) | Unique, Not Null | מזהה כניסה ראשי |
| `is_approved` | Boolean | Default: `false` | האם הפרופיל אושר ע"י מנהל |
| `is_blocked` | Boolean | Default: `false` | האם המשתמש חסום |
| `gender` | Text | `male` / `female` | מגדר |
| `pending_changes` | JSONB | - | שינויים הממתינים לאישור מנהל |
| `is_profile_pending`| Boolean | Default: `false` | האם יש שינויים שממתינים לאישור |
| `is_email_verified` | Boolean | Default: `false` | האם המייל אומת |

## 2. הגדרת זרימת משתמש (User Flow)

### נקודות קצה (Endpoints) ופעולות
- **הרשמה וכניסה:**
    - `POST /register`: יצירת חשבון ראשוני (טלפון, סיסמה, פרטים בסיסיים).
    - `POST /login`: כניסה באמצעות טלפון/מייל וסיסמה.
    - `POST /verify-email`: אימות כתובת מייל באמצעות קוד בן 6 ספרות.
- **פרופיל וחיפוש:**
    - `GET /my-profile`: שליפת נתוני המשתמש (כולל סטטוס אישורי שינויים).
    - `POST /update-profile`: עדכון פרטים רגישים (מצריך אישור מנהל).
    - `POST /update-safe-fields`: עדכון פרטים "בטוחים" (נשמר מיידית).
    - `GET /matches`: מנוע התאמות דו-כיווני (התאמה של "אני מחפש אותו" וגם "הוא מחפש אותי").
- **אינטראקציה:**
    - `POST /connect`: שליחת "לייק" או בקשת קשר ראשונית.
    - `POST /request-photo-access`: בקשה הדדית לצפייה בתמונות.
    - `POST /finalize-connection`: אישור סופי של שני הצדדים למעבר לטיפול שדכנית.

### לוגיקת מעברים וניהול מצבים
- **הצלחה:** ביצוע פעולה (כגון אישור קשר) מעדכן את הסטטוס ב-DB ושולח התראת מערכת + מייל לצד השני.
- **כישלון:** במקרה של דחיית בקשה, הסטטוס משתנה ל-`rejected`. אם אחד הצדדים חוסם/מסתיר, הפרופיל לא יופיע יותר בתוצאות החיפוש.
- **אישור פרופיל:** פרופיל חדש או שינוי פרטים רגישים חייב לעבור אישור ידני של מנהל.

## 3. אפיון טכנולוגי (Tech Stack & Integration)

### רשימת ספריות ותלויות
- **Backend:** Node.js עם Framework מסוג Express.
- **Database:** PostgreSQL (באמצעות ספריית `pg`).
- **Authentication:** JWT (JSON Web Token) לניהול סשנים ו-Bcrypt להצפנת סיסמאות.
- **File Management:** Multer להעלאת תמונות וניהול קבצים מקומי.
- **Security:** Helmet (אבטחת Headers) ו-Rate Limiting (הגנה מפני Brute Force).
- **Communication:** Nodemailer לשליחת מיילים (תומך ב-Ethereal לפיתוח וב-Gmail לפרודקשן).
- **Frontend:** React (Vite) עם שימוש ב-Vanilla CSS (לפי אפיון הקוד).

### ממשקי API חיצוניים
- **SMTP Server:** תקשורת עם שרתי Gmail/Outlook לשליחת התראות.
- **IVR Service (TODO):** תמיכה עתידית בשיחות קוליות לשחזור סיסמה (`/forgot-password`).

## 4. דרישות פונקציונליות (Requirements)

### Core Features (MVP)
1. **הרשמה מאובטחת:** אימות טלפון ומייל.
2. **פרופיל 3 שלבים:** פרטים אישיים, רקע משפחתי, וקריטריוני חיפוש.
3. **מנוע התאמות:** סינון חכם המבוסס על הצלבה של שני הצדדים (גיל, גובה, מגזר, סטטוס ועוד).
4. **ניהול תמונות פרטיות:** התמונות מוסתרות כברירת מחדל וזמינות רק לאחר אישור הדדי.
5. **ממשק אדמין מלא:** אישור פרופילים, ניהול שדכניות, ושיוך תיקים לטיפול.

### Edge Cases (מקרים חריגים)
- **אישור אוטומטי (Auto-Approve):** אם מנהל לא התייחס לבקשת עדכון פרופיל תוך 28 שעות, המערכת מאשרת את השינויים אוטומטית (כדי למנוע תקיעות).
- **ניהול שדות בטוחים:** הפרדה בין שדות שדורשים אישור מנהל (כמו שם, טלפון, סטטוס) לבין שדות שניתן לעדכן חופשי (כמו "על עצמי" או העדפות חיפוש).
- **דיווח על מייל שגוי:** אפשרות לצד ג' שקיבל מייל בטעות להסיר את כתובתו מהמערכת בלחיצת כפתור אחת.
- **שינוי מגדר:** חסימת שינוי מגדר לאחר אישור ראשוני למניעת מניפולציות במנוע החיפוש.
